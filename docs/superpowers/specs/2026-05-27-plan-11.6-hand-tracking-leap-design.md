# WaveConductor v5 — Hand-Tracking Provider + Leap Verification (Plan 11.6)

**Date:** 2026-05-27
**Workstream:** Line parity gate (Plan 11.6 in `docs/superpowers/roadmap.md`)
**Status:** Design — pending Madison review before plan-writing
**Scope window:** ~5–7 days (provider plumbing + HandMesh port + diagnostics UI)

## Goal

Replace the stub `LeaprsProvider` with a real native LeapC binding that streams hand-tracking frames into v5's input pipeline on Madison's macOS dev box (Ultraleap 6.2.0 + OG Leap Motion Controller) and on the Linux/Windows kiosk hardware. Rework the input subsystem to support multiple providers (Leap today, MediaPipe and simulator sources tomorrow), entity-per-hand state so per-hand attached state (HandMesh, future per-hand audio voices) gets Bevy-native lifecycle, and a multi-axis diagnostic status surface that exposes the full leaprs signal — smudged sensor, robust mode, USB faults, dropped frames — that v4's flat 4-state model threw away. Port v4's HandMesh visualization for the Line sketch in the same sprint so the Line `PARITY.md` capture has visual parity end-to-end.

The kiosk install's primary input modality is the Leap Motion Controller. Line cannot ship to the pi-party deployment (per the roadmap) until on-hardware Leap operation is verified end-to-end. This plan is the second of the two manual gates between Plan 11's code-complete tag and the `v5-line-parity` tag.

- README fix: macOS line in the hardware table is incorrect — Ultraleap 6.2.0 with the OG Leap Motion Controller does work on macOS; Madison's dev setup is the proof.
- Soak test re-run with `MockProvider` in the simulator slot emitting steady grab cycles, verifying zero allocations after init in the new system path.
- Hands-on verification on Madison's Mac: pinch (v4 grab actually), release, hold-while-moving, two-hand simultaneous, audio envelope tracking, smudged-sensor detection at the LED dot.
- **Vendored LeapC binaries** under `vendor/leapc/` for all target platforms (macOS arm64 + x86_64, Linux x86_64, Windows x86_64), so cloud CI builds and `cargo build` from a fresh clone Just Work without requiring the Ultraleap SDK to be installed on the build host. ### `ProviderRegistry` vs `ActiveProvider`

Today (Plan 3):

```rust
#[derive(Resource)]
pub struct ActiveProvider {
    pub(crate) inner: Box<dyn HandTrackingProvider>,
}
```

After Plan 11.6:

```rust
#[derive(Resource, Default)]
pub struct ProviderRegistry {
    providers: Vec<RegisteredProvider>,
}

impl ProviderRegistry {
    pub fn register(
        &mut self,
        id: ProviderId,
        role: ProviderRole,
        inner: Box<dyn HandTrackingProvider>,
    ) {
        self.providers.push(RegisteredProvider { id, role, inner });
    }

    pub fn iter(&self) -> impl Iterator<Item = &RegisteredProvider> + '_ {
        self.providers.iter()
    }

    pub fn provider(&self, id: ProviderId) -> Option<&RegisteredProvider> {
        self.providers.iter().find(|p| p.id == id)
    }

    /// Status of the primary provider (or, if none, the first simulator).
    /// What the status LED reads.
    pub fn primary_status(&self) -> ProviderStatus {
        self.providers.iter()
            .find(|p| p.role == ProviderRole::Primary)
            .or_else(|| self.providers.iter().find(|p| p.role == ProviderRole::Simulator))
            .map(|p| p.inner.status())
            .unwrap_or_default()
    }
}

pub struct RegisteredProvider {
    pub id: ProviderId,
    pub role: ProviderRole,
    pub inner: Box<dyn HandTrackingProvider>,
}

/// Identifies a provider. Plan 11.6 only uses `Leap` and `Mock`. The other
/// variants exist so frame `Provenance` and fusion can distinguish providers
/// once future plans land them — the enum's job is forward-compat shape.
#[derive(Clone, Copy, Debug, PartialEq, Eq, Hash)]
pub enum ProviderId {
    Leap,
    Mock,
    WebSocket,
    MediaPipe,
}

#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub enum ProviderRole {
    /// Real hand-tracking source (Leap, MediaPipe). Frames from primary
    /// providers win over simulator frames during fusion.
    Primary,
    /// Synthetic source (mock for tests, mouse-as-hand, recorded playback).
    /// Frames are emitted but lose to any Primary-provided chirality.
    Simulator,
}
```

`ProviderRegistry::new()` is empty. Binary startup populates it via the auto-selection logic. `poll_all_providers` iterates the `providers` Vec, calling `inner.poll()` on each, tagging emitted `HandTrackingFrame`s with the provider's `id`.

### Trait extension

```rust
pub trait HandTrackingProvider: Send + Sync + 'static {
    fn start(&mut self) -> Result<(), HandTrackingError>;
    fn stop(&mut self);
    fn poll(&mut self, now: Duration, out: &mut Messages<HandTrackingFrame>);
    fn status(&self) -> ProviderStatus;
    fn diagnostics(&self) -> ProviderDiagnostics;
}
```

`status()`'s return type changes from a flat `HandTrackingStatus` enum to the multi-axis `ProviderStatus` struct. Old `HandTrackingStatus` is renamed to `PrimaryState` and demoted to a derived view (`ProviderStatus::primary()`).

### `ProviderStatus` shape

```rust
/// Multi-axis snapshot of a provider's lifecycle and health. Updated each
/// poll(). The status LED reads `primary()`; the dev panel reads every field.
#[derive(Debug, Clone, Default)]
pub struct ProviderStatus {
    /// Reachability of the underlying transport (LeapC service for native,
    /// WebSocket endpoint for web).
    pub service: ServiceConnection,
    /// Whether a tracking device is currently attached.
    pub device: DevicePresence,
    /// Bitflags of device-side health conditions. Multiple may be set
    /// simultaneously (e.g., Smudged + Robust). Mirrors leaprs::DeviceStatus.
    pub health: DeviceHealth,
    /// Whether tracking frames are currently flowing, plus heartbeat
    /// diagnostics.
    pub streaming: TrackingFlow,
    /// Bitflags of service-side health conditions. Mirrors leaprs::ServiceState.
    pub service_health: ServiceHealth,
}

#[derive(Debug, Clone, Copy, Default, PartialEq, Eq)]
pub enum ServiceConnection {
    #[default]
    NotStarted,
    /// Service handshake in progress. leaprs `ConnectionStatus::HandshakeIncomplete`.
    Connecting,
    /// Service reached. leaprs `ConnectionStatus::Connected`.
    Connected,
    /// Service not running on this machine. leaprs `ConnectionStatus::NotRunning`.
    ServiceMissing,
    /// Was connected, then dropped. Derived from leaprs `Event::ConnectionLost`.
    Disconnected,
    /// Unrecoverable provider-level error. The reason text lives in
    /// `ProviderDiagnostics::last_error` so this enum stays `Copy`.
    Errored,
}

#[derive(Debug, Clone, Copy, Default, PartialEq, Eq)]
pub enum DevicePresence {
    #[default]
    NoDevice,
    /// A device was attached. The device serial + SDK version live in
    /// `ProviderDiagnostics`; this variant stays unit so the enum is `Copy`.
    Attached,
    /// A previously-attached device was unplugged.
    Lost,
    /// Device reported an error condition (leaprs `Event::DeviceFailure`).
    /// The reason text lives in `ProviderDiagnostics::last_error`.
    Failed,
}

bitflags::bitflags! {
    /// Mirrors leaprs::DeviceStatus 1:1, exposed in our own crate so we don't
    /// leak the leaprs type across the provider trait boundary.
    #[derive(Debug, Clone, Copy, Default, PartialEq, Eq)]
    pub struct DeviceHealth: u32 {
        const STREAMING       = 1 << 0;
        const PAUSED          = 1 << 1;
        const ROBUST          = 1 << 2;  // IR interference, robust mode
        const SMUDGED         = 1 << 3;  // sensor smudged, tracking degraded
        const LOW_RESOURCE    = 1 << 4;
        const UNKNOWN_FAILURE = 1 << 5;
        const BAD_CALIBRATION = 1 << 6;
        const BAD_FIRMWARE    = 1 << 7;
        const BAD_TRANSPORT   = 1 << 8;  // USB issue
        const BAD_CONTROL     = 1 << 9;
    }
}

#[derive(Debug, Clone, Copy, Default)]
pub enum TrackingFlow {
    #[default]
    NotStreaming,
    Streaming {
        /// Time since the last `Event::Tracking` was observed. Updated each
        /// poll() from a monotonic clock.
        last_frame_ago: Duration,
        /// Number of `Event::DroppedFrame` events accumulated since
        /// `start()`. The dev panel can derive a per-second rate from
        /// repeated samples; the provider doesn't compute the rate itself.
        dropped_since_start: u64,
    },
}

bitflags::bitflags! {
    /// Mirrors leaprs::ServiceState 1:1.
    #[derive(Debug, Clone, Copy, Default, PartialEq, Eq)]
    pub struct ServiceHealth: u32 {
        const LOW_FPS_DETECTED        = 1 << 0;
        const POOR_PERFORMANCE_PAUSE  = 1 << 1;
        const TRACKING_ERROR_UNKNOWN  = 1 << 2;
    }
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PrimaryState {
    /// Provider not yet started.
    NotStarted,
    /// Ultraleap service (or WS server, on web) not running.
    ServiceMissing,
    /// Connecting / handshake in progress / was connected but dropped.
    Disconnected,
    /// Service reachable, no Leap device attached.
    ServiceOnly,
    /// Device attached but not currently streaming frames.
    DeviceAttached,
    /// Streaming and the device reports no degraded-health flags. Green.
    Streaming,
    /// Streaming, but `health` contains SMUDGED / ROBUST / LOW_RESOURCE,
    /// or `service_health` contains LOW_FPS_DETECTED /
    /// POOR_PERFORMANCE_PAUSE. Yellow.
    DeviceDegraded,
    /// Device reported a failure (BAD_TRANSPORT / BAD_FIRMWARE /
    /// BAD_CALIBRATION / BAD_CONTROL / UNKNOWN_FAILURE), or
    /// `DevicePresence::Failed`. Red.
    DeviceFailed,
}

impl ProviderStatus {
    /// Collapse the multi-axis state into the single dot-colorable state
    /// the status LED reads.
    pub fn primary(&self) -> PrimaryState { /* match the axes */ }
}

#[derive(Debug, Clone, Default)]
pub struct ProviderDiagnostics {
    /// Device serial number (e.g., "LP00012345"). None on providers that
    /// don't expose it (mock, WebSocket-pre-deviceEvent).
    pub device_serial: Option<String>,
    /// SDK / runtime version string. e.g., "Ultraleap Gemini 6.2.0".
    pub sdk_version: Option<String>,
    /// Currently-active policy flags (e.g., BackgroundFrames). Empty when
    /// no policies are set.
    pub active_policies: Vec<String>,
    /// Number of dropped frames since start(). Same value as
    /// `TrackingFlow::Streaming::dropped_since_start` when streaming;
    /// kept here so the dev panel can show it across all states.
    pub dropped_frames: u64,
    /// Short reason string for the most recent `ServiceConnection::Errored`
    /// or `DevicePresence::Failed`. None when no error has occurred.
    pub last_error: Option<String>,
}
```

### Entity model

```rust
/// Marker component for any currently-tracked hand entity. Spawned by
/// `sync_hand_entities` when a new (provider, raw_id) appears in a fused
/// frame; despawned when that pair disappears from the frame stream.
///
/// Lifetime: from the moment a hand enters the tracking volume until it
/// leaves. May survive across multiple seconds for a stable hand.
#[derive(Component, Debug)]
pub struct TrackedHand;

/// Provider-local stable identifier. Two consecutive frames with the same
/// HandId on the same provider mean "same physical hand". Identifiers may
/// be reused after a hand leaves the tracking volume.
#[derive(Component, Debug, Clone, Copy)]
pub struct HandId(pub u32);

/// Which provider this hand came from, plus the provider-local raw id.
/// Used by `sync_hand_entities` to key the entity table.
#[derive(Component, Debug, Clone, Copy)]
pub struct Provenance {
    pub provider: ProviderId,
    pub raw_id: u32,
}

/// Which hand this is.
#[derive(Component, Debug, Clone, Copy, PartialEq, Eq)]
pub enum Chirality { Left, Right }

/// Palm centroid in Leap-device coordinates (millimeters). Origin: device
/// center. Axes: +X right, +Y up (away from sensor surface), +Z toward
/// the user (with the device's rounded edge facing the user).
#[derive(Component, Debug, Clone, Copy)]
pub struct PalmPosition(pub Vec3);

/// Palm velocity in mm/s.
#[derive(Component, Debug, Clone, Copy)]
pub struct PalmVelocity(pub Vec3);

#[derive(Component, Debug, Clone, Copy)]
pub struct PinchStrength(pub f32);

#[derive(Component, Debug, Clone, Copy)]
pub struct GrabStrength(pub f32);

/// 21-landmark MediaPipe-style layout. Filled by the provider from its
/// native bone/joint data (leaprs digits→bones for Leap, direct landmarks
/// for future MediaPipe).
#[derive(Component, Debug, Clone)]
pub struct Landmarks(pub [Vec3; 21]);

/// 20 bone centers, in finger-then-bone order (5 fingers × 4 bones).
/// Used by HandMesh rendering to position the wireframe spheres. Filled
/// by the provider directly from leaprs::Bone::center() for native;
/// computed as midpoints-between-consecutive-landmarks for providers that
/// don't expose bones (future MediaPipe).
#[derive(Component, Debug, Clone)]
pub struct BoneCenters(pub [Vec3; 20]);
```

### `sync_hand_entities` algorithm

```rust
fn sync_hand_entities(
    mut commands: Commands,
    mut entity_table: Local<HashMap<(ProviderId, u32), Entity>>,
    fused: MessageReader<FusedHandFrame>,
    mut tracked: Query<(&Provenance, &mut PalmPosition, &mut PalmVelocity,
                        &mut PinchStrength, &mut GrabStrength,
                        &mut Landmarks, &mut BoneCenters)>,
) {
    let mut seen_this_frame: HashSet<(ProviderId, u32)> = HashSet::new();
    for frame in fused.read() {
        for hand in &frame.hands {
            let key = (hand.provider, hand.raw_id);
            seen_this_frame.insert(key);
            if let Some(&entity) = entity_table.get(&key) {
                // Update existing entity in place.
                if let Ok(mut bundle) = tracked.get_mut(entity) {
                    // ... write through the fresh values
                }
            } else {
                // Spawn new entity.
                let entity = commands.spawn((
                    TrackedHand,
                    HandId(hand.raw_id),
                    Provenance { provider: hand.provider, raw_id: hand.raw_id },
                    hand.chirality,
                    PalmPosition(hand.palm_position),
                    /* ... */
                )).id();
                entity_table.insert(key, entity);
            }
        }
    }
    // Despawn entities whose hands didn't show up in this frame batch.
    entity_table.retain(|&key, &mut entity| {
        if seen_this_frame.contains(&key) {
            true
        } else {
            commands.entity(entity).despawn();  // recursive — children go too
            false
        }
    });
}
```

The `Local<HashMap<…>>` lives in the system state, not as a resource. It's the only per-(provider, raw_id) bookkeeping; everything else is on the entities.

### Fusion policy (Plan 11.6 is trivial passthrough)

```rust
fn fuse_hand_frames(
    raw: MessageReader<HandTrackingFrame>,
    mut fused: MessageWriter<FusedHandFrame>,
) {
    // Plan 11.6 ships single-provider operation. Passthrough.
    for frame in raw.read() {
        fused.write(FusedHandFrame {
            hands: frame.hands.iter().map(|h| FusedHand {
                provider: frame.provider,
                raw_id: h.id,
                chirality: h.chirality,
                /* copy fields */
            }).collect(),
            timestamp: frame.timestamp,
        });
    }
}
```

Future per-chirality precedence policy (when MediaPipe or simulator lands as a second registered provider):

```text
For each (chirality) ∈ {Left, Right}:
    Among all incoming frames in this tick that contain a hand of `chirality`:
        Prefer ProviderRole::Primary > ProviderRole::Simulator.
        Among Primary providers, prefer Leap > MediaPipe (configurable later).
    Emit at most one FusedHand per chirality.
```

The current Plan-11.6 passthrough is the degenerate case of this rule when only one provider is registered.

### `LeaprsProvider` mapping

leaprs's data model → our data model:

| leaprs | our types | notes |
|---|---|---|
| `Connection::open()` returns a polling loop handle | `LeaprsProvider::start()` opens the connection, transitions `service` from `NotStarted` → `Connecting` → `Connected` | failure to open → `ServiceConnection::ServiceMissing` (when `ConnectionStatus::NotRunning`) or `Errored` |
| `Event::Connection` / `ConnectionLost` | drives `ProviderStatus::service` between `Connected` ↔ `Disconnected` | observed during `poll()` |
| `Event::Device(DeviceRef)` | `ProviderStatus::device = Attached`; capture `device_info.serial` into `ProviderDiagnostics::device_serial` | leaprs's `DeviceRef::open()` is called once on first observation |
| `Event::DeviceLost` / `DeviceFailure` | `ProviderStatus::device = Lost` / `Failed` | |
| `Event::DeviceStatusChange(DeviceStatus)` | merge into `ProviderStatus::health` bitflags; preserves `STREAMING`, `SMUDGED`, `ROBUST`, etc. | mapping is 1:1, bitflag-to-bitflag |
| `Event::Tracking(TrackingEvent)` | one `HandTrackingFrame` emitted; each `Hand` converted to our `Hand` + landmarks + bone centers | also: stamp `TrackingFlow::Streaming::last_frame_ago` = 0, increment internal counter on poll |
| `Event::DroppedFrame` | increment `ProviderStatus::streaming::dropped_since_start` and `ProviderDiagnostics::dropped_frames` | both counters share a value |
| `Event::Policy(PolicyFlag)` | reflect into `ProviderDiagnostics::active_policies` (`BACKGROUND_FRAMES`, etc.) | |
| `leaprs::Hand::digits[i].bones[j]` (5 digits × 4 bones) | `BoneCenters([Vec3; 20])`, order: thumb-meta, thumb-prox, thumb-int, thumb-distal, index-meta, ..., pinky-distal | bone centers via `Bone::center()` (a leaprs method computing midpoint of `prev_joint` and `next_joint`) |
| leaprs joint positions per digit | `Landmarks([Vec3; 21])` in MediaPipe layout: `Wrist`, then for each digit (Thumb, Index, Middle, Ring, Pinky): MCP, PIP, DIP, TIP | wrist = palm-bone start of the index finger's metacarpal (closest analog leaprs gives); thumb has only IP not PIP/DIP, so we copy IP into both PIP and DIP slots to keep the array shape (documented in the conversion code) |
| `leaprs::Hand::pinch_strength` | `PinchStrength(f32)` | already `[0,1]` |
| `leaprs::Hand::grab_strength` | `GrabStrength(f32)` | already `[0,1]` |
| `leaprs::Hand::palm.position` (mm) | `PalmPosition(Vec3)` | leaprs gives mm directly |

### Coordinate projection

```rust
/// V4 parity: top-down ortho onto the Leap (X, Y) plane. Y is height
/// above the device, NOT the user-facing Z axis. v4's empirically-chosen
/// mapping puts hands in the centered 60% of the screen, with 20%
/// deadzones at each edge.
///
/// Ported byte-for-byte from `.worktrees/v4/src/leap/util.ts:115–124`
/// (`mapLeapToThreePosition`).
///
/// # Inputs
/// - `palm_mm`: palm centroid in Leap-device coordinates (mm). Used: x, y.
/// - `window`: viewport size in logical pixels.
///
/// # Output
/// Centered world-space coordinates compatible with v5's mouse_attractor
/// (origin at screen center, +y up, units in logical pixels).
pub fn palm_to_world(palm_mm: Vec3, window: Vec2) -> Vec2 {
    const LEAP_X_HALFRANGE_MM: f32 = 200.0;
    const LEAP_Y_MIN_MM: f32 = 40.0;
    const LEAP_Y_MAX_MM: f32 = 350.0;
    const DEADZONE: f32 = 0.2;
    const USABLE: f32 = 1.0 - 2.0 * DEADZONE; // 0.6

    let x_norm = ((palm_mm.x + LEAP_X_HALFRANGE_MM) / (2.0 * LEAP_X_HALFRANGE_MM))
        .clamp(0.0, 1.0);
    let canvas_x = window.x * (DEADZONE + USABLE * x_norm);
    let world_x = canvas_x - window.x * 0.5;

    // Y is inverted in v4: hand high (350 mm) maps to canvas top (0.2 * h),
    // hand low (40 mm) maps to canvas bottom (0.8 * h).
    let y_norm = ((LEAP_Y_MAX_MM - palm_mm.y) / (LEAP_Y_MAX_MM - LEAP_Y_MIN_MM))
        .clamp(0.0, 1.0);
    let canvas_y = window.y * (DEADZONE + USABLE * y_norm);
    let world_y = -(canvas_y - window.y * 0.5);

    Vec2::new(world_x, world_y)
}
```

Verified at four corners + center against v4 reference: see the unit tests in §Testing.

### Line per-hand attractor

```rust
/// Per-hand attractor power. v4's continuous-power model from
/// `.worktrees/v4/src/particles/leapAttractorPower.ts`:
///
/// ```text
/// if grab > threshold:
///     wanted = grab^1.5 · 5^((-z + 350) / 160)
///     power  = power · (1 - attack) + wanted · attack
/// else:
///     power  = power · decay
///     if powerFloor.is_some_and(|f| power < f) { power = 0 }
/// ```
///
/// Where `grab` is `[0, 1]`, `z` is palm depth in mm (Leap +Z, toward
/// user). Line's `LEAP_POWER_CONFIG` (from `.worktrees/v4/src/sketches/line/index.ts:16-20`):
/// `attack = 0.005`, `decay = 0.5`, `grabThreshold = 0`, no powerFloor.
#[derive(Component, Debug, Default, Clone, Copy)]
pub struct LineHandAttractor {
    pub power: f32,
    pub position: Vec2,
}

pub const LINE_HAND_ATTACK_SPEED: f32 = 0.005;
pub const LINE_HAND_DECAY_SPEED: f32 = 0.5;
pub const LINE_HAND_GRAB_THRESHOLD: f32 = 0.0;
```

The component is added to every `TrackedHand` entity by an `OnAdd<TrackedHand>` observer registered in `LinePlugin::build`, and removed on `OnExit(SketchState::Line)` via a query that detaches the component from all `TrackedHand` entities. (Adding/removing components rather than spawning/despawning entities keeps the hand-tracking entity lifetime owned by the input subsystem, not by per-sketch state.)

Hand-0 (longest-tracked hand — by lowest entity index, since Bevy entities are spawned monotonically) drives `GravityFocalPoint`. Two-hand parity from v4: both hands get their own `LineHandAttractor` and stack into the particle force; no stomping.

Mouse-attractor coexists. The particle stepping system collects from both `MouseAttractorState` (singleton) AND the `LineHandAttractor` query (per-hand) into the same `activeAttractors` array each frame. v4's `LineSketch::step()` does exactly this (`.worktrees/v4/src/sketches/line/index.ts:222–232`).

### HandMesh rendering

Bevy's 2D camera doesn't draw 3D meshes, and v5's Line currently uses `Camera2d` only. v4's HandMesh in Line is bone-center wireframe spheres in the same 3D scene as the particles. v5 reproduces this with a secondary camera and a dedicated render layer:

```rust
/// Marker render layer for hand-mesh geometry. The main Camera2d
/// excludes this layer; the dedicated HandMeshCamera3d targets it.
pub const HAND_MESH_LAYER: RenderLayers = RenderLayers::layer(1);

/// Camera3d that renders only `HAND_MESH_LAYER` entities into the same
/// Hdr view target as Camera2d. Spawned once at LinePlugin startup;
/// despawned in OnExit(Line). Configured with the same Hdr +
/// Tonemapping::AgX + Bloom as the main camera so the hand meshes
/// share the bloom + tonemap pipeline.
#[derive(Component)]
pub struct HandMeshCamera3d;

fn spawn_hand_mesh_camera(mut commands: Commands) {
    commands.spawn((
        HandMeshCamera3d,
        Camera3d::default(),
        Camera { order: 1, ..default() },  // renders after Camera2d (order 0)
        Hdr,
        Tonemapping::AgX,
        Bloom { intensity: 0.15, ..Bloom::NATURAL },
        HAND_MESH_LAYER,
        // OrthographicProjection sized to the window in logical pixels,
        // origin centered, so positions match world-space attractor coords.
        Projection::Orthographic(OrthographicProjection { /* ... */ }),
    ));
}
```

Each bone of each `TrackedHand` is a child `Mesh3d` entity:

```rust
fn spawn_hand_mesh_for(hand: Entity, commands: &mut Commands,
                      meshes: &mut Assets<Mesh>, materials: &mut Assets<StandardMaterial>) {
    let bone_mesh = meshes.add(Sphere::new(10.0).mesh().uv(3, 3));
    let wire = materials.add(StandardMaterial {
        base_color: Color::srgb(0xad as f32 / 255.0,
                                 0xd6 as f32 / 255.0,
                                 0xb6 as f32 / 255.0),
        unlit: true,
        ..default()
    });
    commands.entity(hand).with_children(|p| {
        for i in 0..20 {
            p.spawn((
                Mesh3d(bone_mesh.clone()),
                MeshMaterial3d(wire.clone()),
                Wireframe,                    // bevy/wireframe feature
                WireframeColor { color: Color::srgb(0xad/255., 0xd6/255., 0xb6/255.) },
                HAND_MESH_LAYER,
                BoneIndex(i),
                Transform::default(),
            ));
        }
    });
}
```

A per-frame system `update_hand_mesh_transforms` queries `(&BoneCenters, &Children)` on `TrackedHand` and writes each child's `Transform.translation` from the corresponding bone-center entry. Both cameras share the HDR view target, so bloom + AgX apply uniformly to particles AND hand meshes.

The `bevy/wireframe` feature must be added to the workspace `Cargo.toml`'s `bevy` dependency.

### Provider selection at startup

```rust
// In crates/waveconductor/src/main.rs, after constructing App but before run():

fn install_hand_tracking_providers(app: &mut App) {
    let pref = std::env::var("WAVECONDUCTOR_HAND_PROVIDER")
        .ok()
        .as_deref()
        .unwrap_or("auto")
        .to_lowercase();

    let mut registry = ProviderRegistry::default();

    match pref.as_str() {
        "mock" => {
            registry.register(ProviderId::Mock, ProviderRole::Simulator,
                              Box::new(MockProvider::default()));
            tracing::info!("hand-tracking: forced MockProvider (env override)");
        }
        "leap" | "auto" => {
            let mut leap = LeaprsProvider::default();
            match leap.start() {
                Ok(()) => {
                    registry.register(ProviderId::Leap, ProviderRole::Primary,
                                      Box::new(leap));
                    tracing::info!("hand-tracking: LeaprsProvider started");
                }
                Err(err) if pref == "auto" => {
                    tracing::warn!(?err, "hand-tracking: LeaprsProvider failed to start, falling back to MockProvider");
                    registry.register(ProviderId::Mock, ProviderRole::Simulator,
                                      Box::new(MockProvider::default()));
                }
                Err(err) => {
                    tracing::error!(?err, "hand-tracking: LeaprsProvider failed to start, no fallback (env forced 'leap')");
                    // No provider registered. The app still runs; mouse + touch work.
                }
            }
        }
        other => {
            tracing::warn!(value = %other, "hand-tracking: unknown WAVECONDUCTOR_HAND_PROVIDER value, defaulting to auto");
            // Recurse with "auto" semantics via the same logic.
        }
    }

    app.insert_resource(registry);
}
```

`MockProvider` keeps its current shape (scripted-frame replay), but is repositioned as `ProviderRole::Simulator` instead of being the default. Tests construct their own `ProviderRegistry` with the desired providers; they don't depend on env vars.

### Vendored LeapC binaries

v4's `bin/` archival pattern was driven by the same goal we have now: clone-and-build on any platform without requiring an SDK install. v4 archived Windows `LeapC.dll` plus the WS-bridge executables. v5 with direct `leaprs` FFI to LeapC needs the actual LeapC runtime library on every target platform, not just Windows.

**Layout:**

```
vendor/leapc/
├── LICENSE                          # Ultraleap SDK redistribution terms (copy from SDK installer)
├── README.md                        # SDK version, source, refresh procedure
├── include/                         # C headers (shared across platforms)
│   ├── LeapC.h
│   ├── LeapC.modulemap              # for macOS framework discovery
│   └── ...                          # plus the small set of headers LeapC.h includes
├── macos-aarch64/
│   └── libLeapC.5.dylib             # Apple Silicon, copied from /Library/Application Support/Ultraleap/...
├── macos-x86_64/
│   └── libLeapC.5.dylib             # Intel Mac, same source
├── linux-x86_64/
│   └── libLeapC.so.5                # Ubuntu 22.04, from the Gemini .deb
└── windows-x86_64/
    ├── LeapC.dll                    # Windows, moved here from v4's bin/LeapC.dll
    └── LeapC.lib                    # import library for MSVC linking
```

**Estimated total size:** ~30–50 MB across all platforms. No Git LFS — plain git tracks binaries fine at this scale, and v4's checked-in `LeapC.dll` (13 MB) sets the precedent. If the repo grows past ~200 MB total we'll revisit.

**Refresh procedure:**

`vendor/leapc/README.md` documents:
1. SDK version archived (e.g., "Ultraleap Gemini 6.2.0").
2. Source per platform (which installer / which path the file was copied from).
3. The exact command to refresh from a new SDK release.

This is human-operated, not automated. SDK releases are infrequent (a few times per year), and the kiosk install is pinned to a known-good version.

**License:**

`vendor/leapc/LICENSE` carries the verbatim Ultraleap Enterprise Tracking Licence text. `vendor/leapc/README.md` notes the SDK version and refresh procedure.

**Attribution:**

The licence requires attribution wherever the application is presented. v5 satisfies this in three places:

- A `Credits` / `About` panel entry crediting Ultraleap (Plan 11.5's overlay UI scope; this plan adds the specific text).
- A line in the README's "Acknowledgements" section.
- A line in the kiosk install's `README.txt` shipped alongside the binary.

Exact attribution text lives in `vendor/leapc/ATTRIBUTION.md` for reuse across surfaces.

**Build-script integration:**

`leap-sys` (the unsafe bindings crate `leaprs` depends on) has its own `build.rs` that looks for LeapC in standard SDK install locations. Three options for pointing it at `vendor/leapc/`:

1. **Env-var override** (preferred if `leap-sys` supports it — to be verified): set `LEAPC_SDK_PATH=$CARGO_MANIFEST_DIR/vendor/leapc/$TARGET_PLATFORM` in `.cargo/config.toml`'s `[env]` section.
2. **`[patch.crates-io]` to a vendored leap-sys**: bring `leap-sys` source in-tree (or as a git submodule) with `build.rs` modified to look at `vendor/leapc/`.
3. **Fork `plule/leap-sys` → `madisonrickert/leap-sys`**: point Cargo at the fork's git URL, push our build-script changes there.

The first task in the implementation plan is to confirm which mechanism `leap-sys` exposes and pick the lightest-touch path. Option 1 is the goal; option 3 is the fallback if `leap-sys` requires patching.

**Runtime library loading:**

After `cargo build`, the resulting binary needs to find the LeapC runtime library at execution time. Per-platform mechanism:

| Platform | Mechanism |
|---|---|
| macOS | `@rpath/libLeapC.5.dylib` baked into the binary's load commands; `rpath` set to `@executable_path/../lib` for release bundles, `$CARGO_MANIFEST_DIR/vendor/leapc/macos-{arch}` for dev. Configured via `RUSTFLAGS="-C link-arg=-Wl,-rpath,..."` in `.cargo/config.toml`. |
| Linux | `RUNPATH` baked into the ELF binary, same idea. `-Wl,-rpath,$ORIGIN/../lib` for release; dev path for `cargo run`. |
| Windows | `LeapC.dll` next to the executable, or in any directory on `PATH`. Dev: `target/debug/` gets the DLL copied at build time via a `build.rs` step. Release: bundler copies into the install dir. |

For `cargo run -p waveconductor` during development on Madison's Mac, the binary needs `libLeapC.5.dylib` either:
- Found via `@rpath` resolution pointing at `vendor/leapc/macos-aarch64/` (preferred — works without env vars).
- Or `DYLD_FALLBACK_LIBRARY_PATH=vendor/leapc/macos-aarch64/` set in `.cargo/config.toml`'s `[env]` section (fallback).

**CI implications:**

Cloud builds on GitHub Actions runners (or whichever CI Madison uses) currently can't install the Ultraleap SDK because Ultraleap's installer isn't headless-friendly. With `vendor/leapc/` present, CI just checks out the repo and runs `cargo build` — no SDK install step needed. Existing CI workflows pick this up for free.

**Documentation update:**

`README.md`'s Linux build-prereqs section (currently lists Bevy's apt packages) gets a note that LeapC is vendored in-tree and no separate Ultraleap SDK install is required to build. End-users running the released binary still need the Ultraleap *service* running for the device to be detected — but the build artifact is self-contained.
